HTML

  • HTML stands for HyperText Markup Language.
  • HTML gives content structure.
  • HTML elements are usually composed of an opening tag and a closing tag represented with angle brackets.
  • HTML attributes are always defined in the beginning element, never the closing element.
  • The "head" element contains the metadata for our web application. Metadata is data about our webpage that the browser uses to effectively serve our webpage to users. Metadata is typically not seen by users, but it contains important information about accessibility, search engines, and performance.
  • The "title" element defines the title for a webpage. This element is required and is extremely important for Search Engine Optimization (SEO).
  • The "style" element defines the CSS styles associated with a website.
  • The "base" element defines the base URL for a webpage.
  • The "link" element connects an external resource to the HTML document.
  • The "meta" element defines metadata such as the character set, description, keywords, author, and viewport.
  • The "script" element embeds client-side scripts in the HTML document.
  • "!DOCTYPE" declaration tells the browser what document type it should expect when it is opened.
  • "!DOCTYPE" is not case-sensitive because it is a declaration. Declarations are not case-sensitive; HTML elements are case-sensitive.
  • The "charset" element sets our character encoding.
  • The "http-equiv" element sets an HTTP header for content.
  • The "viewport" element sets the default view for our webpage.
  • The "body" element represents the visible content shown to the user.
  • Semantic HTML elements, the name of the element describes its content and the role it plays on a webpage.
  • The non-semantic element "div" creates a division in the page.
  • The non-semantic element "span" creates an inline division
  • When working with visible HTML elements, the content is contained between an opening tag and a closing tag.
  • The words “element” and “tag” are commonly used interchangeably in the web development world, but it is important to know the difference. An element refers to the entire element from opening tag to closing tag, whereas a tag refers only to what appears inside the angle brackets.
  • Elements that are nested inside other elements are called child elements.
  • The element that holds all the children is known as the parent element.
  • The element "h" plus a number, suchs as "h1", represents the level of heading a given text block represents.
  • the element "p" represents a pragraph or block of text.
  • The elements "ul", "ol", and "li" represent unordered lists, ordered lists, and list items.
  • The "img" element contains information about images that are displayed on the webpage.
  • The "br" element creates a line of empty space, or a line break, between two blocks of content.
  • The anchor element, "a", creates links to the same webpage or other webpages.
  • The "href" attribute points to the URL for the link.
  • The "src" attribute defines the location, or source, of the image file.
  • The "alt" attribute contains a text string that describes the appearance and functionality of the image.
  • The alt text is important for screen readers and other assistive technologies. Alt text can also improve search engine rankings for your website.

CSS

  • CSS stands for Cascading Style Sheet.
  • CSS defines the colors, aesthetics, visual layout, and organization.
  • While it's possible to use define style inline or via internal style sheet, it's easier to update and manage an external style sheet.
    • Inline CSS: used whenn a developer wants to style a specific element directly.
    • Internal Style Sheet: applied to the HTML's 'head' to define rules for a specific page.
    • External Style Sheet: the most common way to manage a webpage's style and linked in the HTML's 'head'.
  • Declarations contain two important components: the CSS property we want to apply and the value of the property.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • To link a CSS File to an HTML File:
    • Make sure the CSS file is saved and you know the file path.
    • Add the following code after the final "meta" element and before the "title" element:
      • link rel="stylesheet" href:"/file_path"
      • In the preceding statement, a "link" element creates the connection to the external file
      • The "rel" attribute specifies the relationship between the current document and the linked document resource.
      • The "href" attribute specifies the location (URL) of the external resource. The href attribute is using a relative path. This means the location is derived from the current location.
    • Save the file.
      • You can use a class selector, such as ".card", to style multiple HTML elements.
      • The period preceding "card" designates this as a class selector. Any HTML element assigned to the class, "card", will gain these property assignments from the CSS.
    • Assigning colors in CSS can be done in many ways, including by semantic references such as "gray" or "blue". We can also refer to a color by its hexadecimal code, or hex #.

Git

  • Git Bash is an application for Microsoft Windows environments which provides an emulation layer for a Git commant line experience.
  • Git Bash Comands:
    • "cd directory_name": navigates to the specified directory.
    • "code ."": opens the current directory in VS code.
    • "pwd": tells you what directory you're currently in.
    • "ls": lists items in current directory.
    • "cd ..": returns to the previous directory.
    • "git add -A": commits all files to the working branch.
    • "git checkout branch_name: return to a specific branch
    • "git checkout -b branch_name": creates a new branch and switches to it.
    • "git commit -m "insert_your_message":
      • We use the "-m" flag to associate a message with our commit. Commit messages should be short descriptive messages of the changes implemented. Once completed, our Git repository is now up to date with our working directory!
    • "git pull origin branch": confirm the current branch is up to date.
    • "git push origina feature/branch_name": push up changes to save.
    • "git status": checks what branch we are currently on.
    • "touch name.file_type": used to create a file type directly from the terminal.
      • examples include:
        • style.css
        • script.js
        • index.html
    • Additional commands can be found here.
  • How to create a feature branch via Git Bash:
    • In the command line, navigate to the desired directory.
    • Type "git status" to check which branch you're currently in. If you're not on the branch you wish, switch to the branch by entering the following Git command: "git checout branch_name".
    • In the command line, pull down the latest version of the branch by entering "git pull origin branch_name".
    • From the branch, use the Git command "git checkout -b feature/new_branch_name" to create a new branch.
    • You should see the following message in your terminal:
      "Switched to a new branch 'feature/new_branch_name'".
    • Confirm you've successfully created and navigated to the new branch by typing "git status" in the terminal. You should see the following message:
      • On branch feature/add-html
        nothing to commit, working tree clean
  • How to save your work to GitHub:
    • Confirm you are on the correct branch.
    • Enter "git add -A" in your terminal to commit all files in the working branch.
    • To commit your changes to your GitHub repository, use the command "git commit -m "your detailed specific message".
      • Expected terminal response:
        [feature/add-html 46a625e] added code to HTML file
        1 file changed, 47 insertions(+)
        create mode 100644 index.html
    • Confirm the current branch is up to date before opening a pull request. Enter the command: "git pull origin branch_name".
      • If the branch is up to date, you'll see the message: "Already up to date."
    • Push the changes to GitHub by using the command: "git push origin feature/branch_name".
  • How to leave the branch in the terminal after a pull request.
    • Enter the command: "git checkout branch_name".
      • Usually, you want to return to the main branch, so the command would be:
        "git checkout main".
        The same applies to the next step.
    • Enter the command: "git pull origin branch_name".

JavaScript

  • JavaScript handles logic and allows users to interact with a website.
  • A variable is a named container that allows us to store data in our code.
  • There are different ways to create a variable. We can use "var" but "const" is more commonly used. Another keyword could be "let".
  • Control flow is the order in which a computer executes code in a script.
  • Note that whereas a CSS file is linked within the "head" element, the JS file is linked within the "body" element.
  • You can view console.log output via Chrome DevTools (F12 on your keyboard).
  • Conditional statements can be made using an IF statement.
  • Conditional statements can be made using a FOR LOOP.
  • When the computer encounters an if statement in a JavaScript file, it will check to see if the condition that we provide is truthy. If it is, then it will do whatever action we ask it to do in between the curly brackets. If the condition isn’t true or has a value that isn’t truthy, then the computer will ignore whatever is in between the curly brackets and move on.
  • In JS, an array is a single variable that is used to hold a group of data.
  • Arrays are useful tools for storing multiple items with different data types in a single variable, but not all arrays must have multiple data types. The array that we will add to our project only contains strings.
  • A function is simply a set of instructions that tells the computer how to perform a certain task.
  • A function's name is followed by parentheses (). Later on, we will define parameters that can be placed inside the parentheses. For now, just remember that it is important to include the parentheses whenever you write a function.
